Changelog
=========

Development
-----------

* `comparison_groups` guards. `savings/model_correction`: a zero comparison-group model magnitude (percent / absolute-percent difference-in-differences) is guarded to scale 0 in both the correction and its variance, so it contributes no correction and stays finite instead of dividing to inf. `individual_meter_matching`: raise `DistanceMatchingError` on an empty comparison pool, and when `max_distance_threshold` filters out every candidate match. `cg_clustering/bounds`: remove the dead always-false `linear` lower-bound branch.
* `clustering` / `comparison_groups` correctness tests (no behavior change). Clustering: extend the shared algorithm harness with `kmedians` / `bisecting_kmedians` and assert ground-truth recovery (ARI) rather than a majority heuristic; parametrize the degenerate-data fallback over the honest-degeneracy family with an explicit spectral forced-k contract; new per-algorithm suites (`k_medians`, `bisect_k_medians`, `dbscan`, `birch`, the vendored `sklearn_bisect_k_means` fork delta, `affinity`, `spectral_divisive`, `parallel_analysis`, `clustering_settings`); validity-index monotonicity-in-separation and hand-computed / sklearn-reference values; `has_cluster_structure` grouped-agreement branch logic; DWT roundtrip; voter-discriminability, `prepare_labels`, k-penalty, and the `cluster_features` row-order contract. Comparison groups: IMM matches equal brute-force nearest neighbours; direct `t_and_ks_test`; IMM / HiGHS settings validators; base-class `get_loadshapes`-before-fit contract; a deliberately-mismatched population that fails stratified equivalence; and a `model_correction` regression snapshot across hourly / daily / billing × ODID / PCTDID / ABSPCTDID, pinned on real ComStock-derived inputs (committed `.npz` fixtures generated by `tests/comparison_groups/savings/generate_correction_fixtures.py`).
* Test layout: reorganize the test tree to mirror the source package (no behavior change). Move the top-level `test_caltrack_*` / `test_segmentation` / `test_derivatives` into `tests/eemeter/hourly_caltrack/`; `test_features` / `test_transform` / `test_exceptions` / `test_warnings` into `tests/eemeter/common/`; the `common/stats` transform/loss/outlier tests (and the misplaced `test_adaptive_loss`) into `tests/common/stats/`; and the billing tests out of `tests/eemeter/daily_model/` into `tests/eemeter/billing_model/`. Snapshots travel with their tests. Make `tests/eemeter/` a package so its subpackages namespace under `eemeter.*` (avoids a `common` basename collision with `tests/common/`). In `test_derivatives.py`, the session fixtures now `.copy()` the shared `comstock` data before constructing `Daily`/`Billing` data objects, so the new alphabetical collection order can't leak a tz-converted frame into the design-matrix tests; the in-place mutation in the data classes themselves is tracked separately. Drop `tests/test_version.py` (a bare `__version__` major-version assertion — no behavioral coverage).
* CI: the `update-win-snapshots` workflow is now `workflow_dispatch`-only (was per-PR, where auto-regenerating would have blessed Windows output rather than pinning a baseline). Fix its artifact glob to `*_win*.json` (the prior `**/*_win.json` never matched the `[regression_win]` snapshot names, so it committed nothing) and repoint the `test_derivatives` snapshots to `tests/eemeter/hourly_caltrack/__snapshots__/`.
* Test/coverage infrastructure. Collect the previously-orphaned `tests/common/metrics.py` (renamed to `test_metrics.py` — its `test_acf` never ran because the filename lacked the `test_` prefix) and correct its stale expectations to `acf`'s actual per-method output. Add a `tox -e coverage` environment that runs serially under `NUMBA_DISABLE_JIT=1` so `@numba.jit` functions are visible to the coverage tracer (they execute as native code and report as uncovered otherwise); add `[tool.coverage.report]` `exclude_lines` and omit the `comparison_groups/savings/dev/` work-in-progress scratch. An `xfail` records that `acf(ac_type=STATIONARY_STATS_FFT)` omits zero-padding and so returns the circular (not linear) autocorrelation.
* `common/hourly_interpolation`: `_autocorr_fcn` computes the autocorrelation via FFT above 16 lags and the masked loop at or below, returning identical values (missing data is zero-filled after centering, so it contributes 0 to each lag product). Remove the unused `_autocorr_fcn2` / `_autocorr_fcn3` reference variants; raise on an unknown backup interpolation method. Add direct characterization tests pinning both branches to the original output.
* Bug fix (`clustering`): degenerate-data handling across the selection and algorithm layers.
  - `ClusteringResult`: `.k` / `.metrics` / `.labels` raised `IndexError` when every candidate labeling was rejected by `prepare_labels` (empty `_labels_store`, e.g. all collapsed below `n_cluster_lower`). They now raise a clear `ValueError` naming `n_cluster_lower` when `k=1` is disallowed, and return the single cluster when it is allowed.
  - `ClusteringResult`: when valid candidates exist but the council produces no scored winner, the fallback returns the labeling with the fewest clusters (`>= n_cluster_lower`) instead of the insertion-order-first one, and never a count below the lower bound.
  - `ClusteringResult`: a single specified k (one distinct `k`) bypasses the gap gate and cross-k council entirely, returning the within-k best with full confidence.
  - `SingleKMetrics`: add `unique_valid_labels` / `valid_label_count` (outlier label `-1` excluded); `ClusteringResult.k` uses the valid count.
  - `bisect_k_medians`: bound the per-node split retries so unsplittable data (e.g. identical points) no longer loops forever — after the retry budget the node is accepted as a terminal cluster.
  - `spectral`: the sparse self-tuning path falls back to a dense `eigh` of the smallest eigenpairs when ARPACK fails to converge (previously re-raised).
* `comparison_groups` test suite: coverage across the package — `Data` ingestion (time-series and features-only), the `Comparison_Group_Algorithm` base, the public `CG_Clustering` / `Individual_Meter_Matching` / `Stratified_Sampling` / `random_sampling` flows, cluster bounds, `savings` correction-cap and settings validators and diagnostics, and equivalence / bin-count edge cases. Per-subpackage test directories made packages to avoid duplicate-basename collisions.
* `comparison_groups/stratified_sampling`: modernized to match the rest of the package — single linear settings-driven `get_comparison_group` orchestration; standardize on the shared `id` column (was `meter_id`); fold the module-level constants into settings; expose diagnostics as methods on `Stratified_Sampling`; render diagnostic plots with matplotlib instead of plotnine; delete the dead `param_selection` / `const` modules and modernize class declarations. Output is pinned identical by new snapshot regression tests over the sampled comparison group.
* `clustering`: use skfda `FourierBasis` (was the deprecated `Fourier`); suppress the invalid-log warning in the cluster-bound fallback.
* `comparison_groups`: correct `min_n_sampled_to_n_treatment_ratio` docstrings to float; harden `ids_to_index` against duplicate ids and guard an empty bin count; validate non-positive `n_meters` in `random_sampling`.
* Bug fix (`comparison_groups`): `cg_clustering.treatment_fit` read the removed `settings.normalize`; now reads `settings.feature_transform.normalize`.
* Bug fix (`comparison_groups/common`): `Data` loadshape-type / time-period reads use the lowercase attributes (`loadshape_type`, `time_period`); the standalone hour period is keyed `TimePeriod.HOUR` in `time_period_row_counts` / `min_granularity_per_time_period` (was the mismatched `'hourly'` → `KeyError`); a features-only `Data` no longer crashes when `loadshape_df` is `None`.
* `comparison_groups`: comparison-pool trimming uses a local seedable `np.random.RandomState(settings.seed)` instead of the unseeded global; adds `Data_Settings.seed` (default `None`).
* Bug fix (`comparison_groups/cg_clustering`): the initial cluster-weight guess gave an exact-match treatment (distance 0) zero weight via a divide; it now captures the exact-match mask before the divide and assigns weight 1.0.
* Bug fix (`comparison_groups/savings/model_correction`): index cluster outputs and `T_weight` by enumeration position, not label value (non-contiguous / float labels raised `IndexError`); the algorithm-`None` exit returns a per-CG-meter boolean mask (was a 0-d scalar); uncertainty uses the absolute form `scale**2*CG_diff_var + CG_diff**2*scale_var`, which stays finite when `CG_diff == 0` (the relative form divided by `CG_diff**2` → NaN); an all-zero model-magnitude cluster falls back to uniform weights.
* Bug fix (`comparison_groups/individual_meter_matching`): no-duplicate matching uses greedy assignment with a full-pool fallback; the pool prefilter runs per-treatment in chunks; warns when `n_match` is capped below the requested value.
* Bug fix (`comparison_groups/stratified_sampling`): `chisquare_dist` guards the 0/0 quantile-mean term (was NaN into bin selection); the sampled-to-treatment ratio is no longer floored; bin bounds are validated.
* `comparison_groups/savings`: move work-in-progress scratch files into `savings/dev/`.
* Python 3.13 and 3.14 support: add the classifiers and extend the tox/CI matrix. Bump `nlopt` to `>=2.10` (first release with cp313/cp314 wheels — the prior unpinned `nlopt` failed to build on 3.14). CI architectures are unchanged: `nlopt` and `fdasrsf` publish wheels only for linux-x86_64, macOS-arm64, and win-amd64, which the runner matrix already covers. Add a `build (all)` fan-in job so branch protection can require the whole matrix through one status check.
* Top-level cleanup. Remove dead files (`setup.cfg`, `MANIFEST.in`, `bump_version.sh`, `.gitattributes`) and the orphaned `docs/gridmeter/` autodoc stubs (the docs site lives in the `opendsm-website` repo). Fold `pytest.ini` and `.coveragerc` into `pyproject.toml`, and set the coverage `sys.monitoring` core (`core = "sysmon"`) so `--cov` no longer disables the debugger on Python 3.12+. Replace the stale `snapshottest==0.6.0` / `typing` dev pins with `syrupy>=5,<6` (the suite already uses syrupy; the old pin crashed `pip install .[dev]` on 3.12+). Add a PEP 561 `py.typed` marker so type information ships in the wheel. Fix the docker-compose dev-env (rename image to `opendsm`, drop the broken `jupyter`/`docs` services and the hardcoded `platform`). Update CONTRIBUTING and the PR template: the API reference auto-generates from google-style docstrings via the `opendsm-website` repo, and the package version is set in `pyproject.toml`.
* Tutorial-data loading. Remove the redundant, broken `load_tutorial_data` (no remaining callers) in favor of `load_test_data`. `load_test_data` now resolves files in order: the in-repo `data/` copy (source checkouts, CI), then a user cache directory, then a one-time download from the OpenDSM repository — so installed users no longer write into `site-packages` or re-download every call. The treatment dataset is `hourly_data_0.parquet` (renamed from `hourly_data_2.parquet`); comparison-group data (`hourly_data_1`/`hourly_data_2`) raises a clear "not yet available" error until those files are added. `data/` is excluded from the sdist (it is download-on-demand, never shipped).
* Packaging metadata: remove a duplicated `scikit-learn` dependency, declare the license with the SPDX `license = "Apache-2.0"` field (PEP 639) instead of the deprecated table form, use `https` project URLs, and remove the `opendsm` console-script entry point (it pointed at a nonexistent `opendsm.cli`, so the installed command always failed).
* `metrics.py`: `BaselineMetrics.pearson_r` returns `1.0` when observed and predicted are identical, and `acf` (MOVING_STATS) returns `1.0` for identical lag slices — both avoid scipy/numpy constant-input warnings and `nan`. `EEMeterWarning.data` is now optional (`Optional[Union[dict, list]] = None`).
* Sufficiency criteria + data settings: unify the four `*SufficiencySettings` classes (`Temperature`/`Ghi`/`Observed`/`Joint`) onto a common `ColumnSufficiencySettings` whose coverage fields are `Optional` and default to `None`, where `None` means the check does not apply to that column. Per-column active-check defaults are unchanged (Temperature: hourly/daily/monthly/period; GHI: monthly; Observed: hourly/daily/monthly/unique; Joint: daily). `SufficiencyCriteria` replaces the per-column if/elif ladders with `_should_skip_col` / `_col_settings` / `_col_display_name` helpers; `_check_unique_values` is generalized to any column (still observed-only by default). Two behavior changes: (1) `_check_negative_observed_values` now runs on reporting-period gas data (previously skipped for all reporting data; still skipped for electricity); (2) the `insufficient_unique_observed_values` warning payload is now `{n_unique_values, n_total_values, unique_percentage, min_required_percentage}`. Float values in disqualification/warning payloads are rounded to 4 significant figures.
* `distribution_transform`: class-based API for all transforms. New `Standardize`, `Bisymlog`, `YeoJohnson`, `BoxCox` classes (all in `opendsm/common/stats/distribution_transform/`) with `fit` / `transform` / `inverse_transform` / `to_dict` / `from_dict`. `YeoJohnson(robust=True)` implements the Raymaekers & Rousseeuw 2021 robust algorithm; `YeoJohnson(robust=False)` wraps scipy's `yeojohnson`. `BoxCox` is new (robust and non-robust variants). Removes the old `scipy_yeo_johnson.py` and `raymaekers_robust_yeo_johnson.py` modules. `OoM` in `opendsm/common/utils.py` now wraps with `np.atleast_1d` so it handles scalar inputs, and `OoM_numba` returns a `float64` array so `method="exact"` is no longer truncated to whole numbers for integer input (round/floor/ceil are unaffected — their values are already whole numbers). **Breaking change** for users who set `OutlierRejectionSettings.transform` directly: `TransformChoice` enum values rename to `YEO_JOHNSON` / `ROBUST_YEO_JOHNSON` / `BOX_COX` / `ROBUST_BOX_COX` (old `SCIPY_YJ` / `ROBUST_SCIPY_YJ` / `ROBUST_YJ` removed); default remains `None`, so users who haven't opted into outlier rejection are unaffected. Tests added under `tests/common/test_{standardize,bisymlog,yeo_johnson,box_cox}.py`.
* `adaptive_loss`: replace `scipy.optimize.minimize_scalar` and the grid-search `_fast_weighted_alpha` with a numba-compiled bounded Brent (`_numba_brent_alpha`), plus numba-compiled penalized loss and loss weights (`_numba_penalized_loss`, `_numba_loss_weights`) and a De Boor B-spline evaluation of `ln_Z` (`ln_Z_numba`). The alpha optimization now runs entirely inside numba — no Python↔numba dispatch overhead per iteration, no 20-candidate grid-search precision loss. Bit-exact (within rounding) match to the prior scipy result on uniform and kernel-weighted inputs; no snapshot regenerations required. Deletes the now-unused `_fast_weighted_alpha` and `kernel_adaptive_weights`.
* Regression-test snapshots: replace per-element `predicted_values` arrays with a compact `regression_block` covering predicted-distribution aggregates, residual aggregates, `pearson_r`, structural counts, a frequency-specific time lens (season × hour-of-week for hourly, season × day-of-week for daily, season for billing), temperature bins, and GHI bins (when present). New `tests/regression_metrics.py` helper; converted `test_hourly_model_regression.py`, `test_daily_model_regression.py`, `test_billing_model_regression.py`, `test_caltrack_hourly_regression.py`, and the regression-marked tests in `test_derivatives.py`. The cell-wise lenses average over many points and survive cross-platform BLAS/FMA reordering. `FLOAT_ATOL` raised to `2e-6` to cover bin-mean residuals near zero where rtol can't help. The four daily-model-derived tests pin Windows-specific snapshots (via `SNAP_SUFFIX = "_win"`) because the SBPLX optimizer converges to a different local minimum on win32; a manual `update-win-snapshots` workflow regenerates them.
* HourlyModel: per-hour effective degrees of freedom in uncertainty. `_set_baseline_metrics` now receives `X_fit` and computes per-hour edf via Zou-Hastie SVD closed form on the active set; each per-hour `BaselineMetrics.num_model_params` becomes `round(edf_h)` clamped so `ddof >= 3`. `_calculate_predicted_uncertianty` dispatches to a per-hour path when per-hour baseline metrics are present, otherwise to the legacy global edf.
* Bug fix (HourlyModel): per-hour edf `λ₂` scales by the elastic-net fit-sample count (fit days), not the hourly row count.
* Bug fix (HourlyModel): `correct_missing_temporal_clusters` compares temporal-cluster indices with `Index.equals()` instead of `==`, which raised on unequal-length indices when predicting a period missing a temporal cluster.
* Bug fix: `t_stat` now takes degrees of freedom directly (sample-size callers — `unc_factor`, caltrack `wrapper.py` — pass `n-1`); `ReportingMetrics.t_stat` passes baseline `ddof`, fixing a double degree-of-freedom subtraction in its uncertainty t-quantile.
* Bug fix (all models): `_calculate_uncertainty` paths pass `confidence_level=1-uncertainty_alpha`. `uncertainty_alpha` is documented as a significance level; previously it was passed directly as a confidence level, double-complementing and producing a ~13x understatement of the t-quantile.
* HourlyModel: split `eemeter/models/hourly/model.py` into focused modules. New `regressors.py` hosts `SafeLinearRegression`, `SafeRidge`, `SafeLasso`, `SafeElasticNet`, and `AdaptiveElasticNetRegressor` (all via a `_NullFitMixin` that short-circuits `fit()` when `y` is all zeros). New `scalers.py` hosts `SafeStandardScaler` and `SafeRobustScaler` (clamp near-zero `scale_` to 1.0 to avoid blow-up on near-constant features).
* HourlyModel: restore `BaseHourlySettings.uncertainty_scale_factor` (default 1.0). Used by the ASHRAE-14 uncertainty path as a final multiplicative coverage-calibration factor — set higher than 1.0 post-deployment if observed coverage runs below nominal.
* `_HourlyData` settings now stored on a private `_settings` attribute with a read-only public property; constructor also accepts a `HourlyDataSettings` instance directly (in addition to `dict` and `None`) and raises `TypeError` for other inputs. Non-nanosecond timezone-aware datetime conversion no longer round-trips through a string cast that lost timezone info on some pandas versions.
* Test suite: `tests/eemeter/hourly_model/test_hourly_data.py` — 162 cases covering `HourlyBaselineData` / `HourlyReportingData` initialization, the data-processing pipeline, all `HourlySufficiencyCriteria` checks, and integration/edge cases. Synthetic-data generators added to `tests/eemeter/hourly_model/conftest.py`.
* Clustering reorganization: `opendsm/common/clustering/` split into `algorithms/`, `metrics/`, and `transform/` subpackages. New algorithms (`bisect_k_medians`, `k_medians`), spectral split into `spectral/` subdir (spectral, spectral_divisive, _affinity), per-algorithm `settings.py` and `protocol.py`. Metrics split into `cross_k_metrics`, `dbcv`, `label_ops`, `labels`, `selection`, `settings`, `single_k_metrics`, `voting`. Transform module is now a directory (`fpca`, `normalize`, `parallel_analysis`, `wavelet`, `settings`, `transform`). `scoring.py` and `voting.py` removed (folded into metrics).
* `eemeter/models/hourly/settings.py`: `ClusteringSettings` factory pins the pre-reorg defaults explicitly (spectral algorithm + scoring weights, wavelet PCA in "global" scope with `pca_n_components="mle"` and `variance_weighted=False`, magnitude features disabled). With those pins, aggregate hourly predictions match master to within 1e-5 relative on sum/mean. **Per-hour spread shifts modestly** — max rel ~3%, min rel up to 12%, std rel ~1% — because some new wavelet/PCA internals (per-band auto-cap when subbands fall below 4 coefficients, weighted magnitude-feature normalization that has no master analogue) cannot be pinned from the factory. Hourly regression snapshots regenerated to reflect the new per-hour values.
* HDBSCAN wrapper now passes `copy=True` to sklearn so it doesn't mutate caller arrays (matches sklearn's future default).
* `_check_self_tuning_gamma` compares `gamma` against its default instead of `model_fields_set`, so serialized settings round-trip cleanly.
* `FeatureMagnitudeSettings.features` default uses enum members instead of raw strings, silencing pydantic serializer warnings.
* IMM memory optimization: `comparison_groups/individual_meter_matching/distance_calc_selection.py` processes treatments in chunks instead of materializing the full N_treat x N_pool matrix; matching settings updated to expose the new path.
* `_autocorr_fcn` (in `opendsm/common/hourly_interpolation.py`): fill fully-masked lag windows to 0.0 via `ma.filled`. Previously, `ma.sum` over an all-masked input returned a masked scalar that became NaN at assignment, and the downstream `argpartition`+reverse sort placed NaN-correlations at the front of the "top correlations" list, causing spurious lag selections and NaN-poisoned fills on gappy meter data. Drop the orphan `warnings.catch_warnings` block whose scope didn't include the line that emits the warning.
* Sufficiency: new `ObservedSufficiencySettings.min_pct_unique_values` (default 0.10) and matching `SufficiencyCriteria._check_unique_values`, applied in daily and hourly baseline checks. Models trained on near-constant observed data (<10% unique values) are now disqualified with a `eemeter.sufficiency_criteria.insufficient_unique_observed_values` warning.
* `generalized_loss_weights`: weight negative and positive residuals symmetrically by magnitude (previously only positive residuals were downweighted)
* `adaptive_loss`: kernel-localized adaptive weighting (`kernel_adaptive_weights`, `KernelWeightCache`, `_fast_weighted_alpha`) so scale and shape of the generalized loss vary smoothly across the temperature axis; optional `obs_weights` for `adaptive_loss_fcn`
* `ellipsoid_test`: improved split filter; daily-model optimizer no longer errors when `initial_step` is `None`
* `daily/plot.py`: plot function accepts `ax`, `include_scatter`, `model_color`, `include_uncertainty`; returns axes; reworked normalize-axes logic
* `mu_sigma`: scope numpy error policy via `np.errstate` instead of `np.seterr` so `'raise'` no longer leaks into every subsequent caller in the process
* HourlyModel: propagate `settings.seed` to `temporal_cluster.wavelet_transform` and `temporal_cluster.fpca_transform` so all stochastic components inherit the same seed (fits are now reproducible end-to-end when `seed` is set)
* Migrate test suite from `eemeter.samples` (bundled IL/UK CSVs) to ComStock fixtures backed by `hourly_data_2.parquet`; mark 64 IL-data-shape-dependent assertions as skipped pending rewrite
* Add per-model snapshot regression tests (daily, billing, hourly solar + non-solar, caltrack hourly) pinned via `syrupy`
* Replace unmaintained `snapshottest` dev dependency with `syrupy>=5,<6`; add `TolerantJSONSnapshotExtension` using `math.isclose(atol=1e-10, rtol=1e-10)` for cross-machine BLAS portability
* Delete `eemeter.samples` and `eemeter.utilities.io` (no remaining consumers)
* Speed up test suite via session-scoped fixtures and `slow`/`regression`/`hourly_model` pytest markers; fast loop with `pytest -m "not slow"`
* `test_data.py`: scope bare `except` to `OSError`, drop unreachable debug `raise`
* `test_distance_match_large_treatments`: skip on hosts with <12 GB RAM
